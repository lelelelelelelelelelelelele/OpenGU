## 方法分类表（按代码框架）

### A) Shard / Partition-based（GraphEraser / GraphRevoker）
- **核心思想（一句话）**：先把训练图切成多个 shard，只重训受影响 shard，再聚合输出，近似替代全量重训。  
- **论文直觉**：把“删除请求的影响范围”局部化，利用 partition 降低 unlearning 开销；代价是跨 shard 信息泄漏可能让遗忘不彻底。  
- **代码对应**：抽象骨架在 `pipeline/Shard_based_pipeline.py:141`；GraphEraser 在 `unlearning/unlearning_methods/GraphEraser/grapheraser.py:24`。  
- **适用任务**：node/edge/feature unlearning（GraphEraser 分支见 `grapheraser.py:282`、`grapheraser.py:295`、`grapheraser.py:306`）；downstream 支持 node/edge/graph（`grapheraser.py:323`、`grapheraser.py:332`）。  
- **典型实现套路**：pipeline 负责编排（partition/train/unlearn）；method 类负责“怎么切图、怎么更新 shard、怎么聚合”；trainer 负责每个 shard 的优化与推理（GraphEraser 用 `get_trainer`，`grapheraser.py:171`）。  
- **关键细节 x3**：  
  1) 分区缓存复用：`community/shard/train_graph/train_data` 都会落盘并复用（`config.py:31`、`utils/dataset_utils.py:761`、`utils/dataset_utils.py:867`）。  
  2) 聚合不是简单平均：可选 `mean/optimal/majority/contrastive`（`aggregator.py:121`），`optimal` 还会学习 shard 权重（`aggregator.py:167`）。  
  3) 遗忘不彻底风险：默认只重训 `affected_shard`（`grapheraser.py:712`），未受影响 shard 参数保留。  
- **最短阅读路径**：`pipeline/Shard_based_pipeline.py:91` → `unlearning/unlearning_methods/GraphEraser/grapheraser.py:51` → `.../partition/graph_partition.py:21` → `.../aggregation/aggregator.py:31`。  

### B) IF / Certified / Approximation family（GIF / CEU / CGU / GST / IDEA / ScaleGUN）
- **核心思想（一句话）**：不（完全）重训，直接估计“删除样本后参数应如何变化”，用影响函数/Hessian 近似或认证边界做快速更新。  
- **论文直觉**：把 unlearning 写成参数扰动估计问题，重点是速度与理论保证（近似误差 vs 可认证性）。  
- **代码对应**：基类 `pipeline/IF_based_pipeline.py:7`；GIF `unlearning/unlearning_methods/GIF/gif.py:21`；CEU `.../CEU/ceu.py:25`；CGU `.../CGU/cgu.py:71`；IDEA `.../IDEA/idea.py:16`；GST 为独立类 `.../GST/gst_based.py:24`。  
- **适用任务**：node/edge/feature 都有实现分支（如 GIF `gif.py:607`、`gif.py:623`、`gif.py:639`；CEU `ceu.py:309`、`ceu.py:316`）。downstream 常见 node/edge，部分支持 graph（如 GIF `gif.py:592`、`gif.py:646`）。  
- **典型实现套路**：pipeline 管 run/metric；method 负责 unlearning request + 近似更新；trainer（如 `GIFTrainer`）负责 `eval_unlearn/get_loss`（`task/GIFTrainer.py:48`、`task/GIFTrainer.py:115`）。  
- **关键细节 x3**：  
  1) 近似核心在 Hessian-vector product：GIF `hvps` + 递推 `h_estimate`（`gif.py:760`、`gif.py:773`、`gif.py:784`）。  
  2) 复用预训练软标签做 MIA 与效果对比（`gif.py:130`、`gif.py:185`）。  
  3) 近似误差来源：k-hop influenced set 截断（`gif.py:685`）+ 固定 iteration/scale（`gif.py:761`）。  
- **最短阅读路径**：`pipeline/IF_based_pipeline.py:107` → `unlearning/unlearning_methods/GIF/gif.py:585` → `gif.py:407` → `gif.py:750` → `task/GIFTrainer.py:48`。  

### C) Learning / Distillation-based（GNNDelete / MEGU / GUKD / D2DGN / SGU / UTU）
- **核心思想（一句话）**：把“遗忘”当成一个可学习目标，通过蒸馏、删除模块、门控或局部再训练来逼近 retrain。  
- **论文直觉**：不追求严格 IF 推导，而是设计可训练的 forget objective，在效果-效率间做经验最优。  
- **代码对应**：基类 `pipeline/Learning_based_pipeline.py:6`；GNNDelete `.../GNNDelete/gnndelete.py:28`；MEGU `.../MEGU/megu.py:33`；SGU `.../SGU/sgu.py:32`；UTU 是独立类 `.../UTU/utu.py:15`。  
- **适用任务**：多数覆盖 node/edge/feature（如 GNNDelete `gnndelete.py:102`）；UTU 更偏 edge/link（类注释 `utu.py:17`）。  
- **典型实现套路**：pipeline 统一 run；method 设置 `unlearn_trainer` 并组织数据；专用 trainer 实现具体损失和删除模块训练（如 GNNDelete 设 `GNNDeleteTrainer`，`gnndelete.py:75`）。  
- **关键细节 x3**：  
  1) 预处理缓存对速度很关键（如 GNNDelete 的 `features_pre`，`gnndelete.py:65`）。  
  2) 常见“局部参数更新而非全模型重训”，有残留记忆风险。  
  3) 评估常混合 utility 与 privacy（F1/AUC/MIA），但不同方法触发条件不完全一致。  
- **最短阅读路径**：`pipeline/Learning_based_pipeline.py:80` → `unlearning/unlearning_methods/GNNDelete/gnndelete.py:57` → `task/GNNDeleteTrainer.py`（训练与 MIA 细节）→ `attack/Attack_methods/GNNDelete_MIA.py:4`。  

## 三种代表方法 walkthrough（按调用顺序）

### 1) GraphEraser（partition-based）
**为什么选它**：它最完整体现“分区-重训-聚合”范式，也是 shard 家族核心实现。

1. `run_exp()` 入口在基类：`pipeline/Shard_based_pipeline.py:91`。  
   输入是 `args + model_zoo.data`；`args["num_runs"]` 决定循环次数（`Shard_based_pipeline.py:112`）。  
2. **Partition 阶段** `exp_partition()`（`Shard_based_pipeline.py:141`）调用：  
   - `gen_train_graph()`：切 train graph 并保存 `train_data/train_graph`（`grapheraser.py:51`、`grapheraser.py:77`）。  
   - `graph_partition()`：按 `partition_method` 选策略（`grapheraser.py:89`，`graph_partition.py:24`）。  
   - `generate_shard_data()`：构造每个 shard 的 `Data` 并 `save_shard_data`（`grapheraser.py:110`、`grapheraser.py:148`）。  
3. **Train 阶段** `exp_train()`（`Shard_based_pipeline.py:155`）：  
   - `load_data()` 回读 shard/train_data（`grapheraser.py:158`）。  
   - `determine_target_model()` 用 trainer（默认 `BaseTrainer`，`grapheraser.py:171`）。  
   - `train_target_models()` 逐 shard 训练并 `save_target_model`（`grapheraser.py:964`、`grapheraser.py:992`）。  
   - `aggregate()` 通过 `Aggregator.generate_posterior()+aggregate()` 得到主指标（`grapheraser.py:996`）。  
4. **Unlearn 阶段** `exp_unlearn()`（`Shard_based_pipeline.py:168`）：  
   - `generate_requests()` 从 `data/unlearning_task/...txt` 读取或生成删除目标（`grapheraser.py:275`、`grapheraser.py:283`）。  
   - `unlearn()` 分派 node/edge/feature 三类响应函数（`grapheraser.py:318`）。  
   - 关键中间态：`unlearned_train_graph/train_data/shard_data` 写入 `config.unlearned_file_*`（`dataset_utils.py:789`）。  
5. **输出指标**：  
   - effectiveness：`average_f1`（`grapheraser.py:330`）。  
   - efficiency：`avg_partition_time/avg_training_time/avg_unlearning_time`（`Shard_based_pipeline.py:66`、`grapheraser.py:342`）。  
   - robustness（poison setting）：`poison_f1`（`grapheraser.py:272`）。  
   - privacy（MIA）：类里有 `attack_graph_unlearning()`（`grapheraser.py:381`），但默认 `exp_unlearn()` 没调用 `attack_unlearning()`（`Shard_based_pipeline.py:175` 被注释）。  

### 2) GIF（IF-based）
**为什么选它**：它把 IF 近似、k-hop 影响域、MIA 三块都串起来，最能体现“近似遗忘”代码形态。

1. `run_exp()` 入口在 `pipeline/IF_based_pipeline.py:107`。每轮调用：`determine_target_model → train_original_model → unlearning_request → unlearn → (可选) mia_attack`。  
2. `determine_target_model()`：强制 `unlearn_trainer=GIFTrainer`（`gif.py:355`、`gif.py:362`），实际训练/评估落在 `task/GIFTrainer.py`。  
3. `train_original_model()`：先训练原模型，再记录 `avg_training_time`，edge-poison 配置下记录 `poison_f1`（`gif.py:379`、`gif.py:402`、`gif.py:404`）。  
4. `unlearning_request()` 输入是 `args + data.train_indices + unlearning txt`：  
   - 读取/采样 unlearning nodes/edges（`gif.py:607`、`gif.py:623`）。  
   - 构造 `edge_index_unlearn/x_unlearn`（`gif.py:599`、`gif.py:603`、`gif.py:621`、`gif.py:645`）。  
   - 计算受影响 k-hop 节点/边（`gif.py:646`、`gif.py:685`、`gif.py:711`）。  
5. `unlearn()`：  
   - `get_if_grad()` 得到 `(grad_all, grad1, grad2)`（`gif.py:407`）。  
   - `approxi()` 用 Hessian-vector recursion 估计参数改变量（`gif.py:750`、`gif.py:773`、`gif.py:775`）。  
   - 调 `GIFTrainer.eval_unlearn()` 直接评估“近似后参数”效果（`gif.py:781`、`task/GIFTrainer.py:48`）。  
6. `mia_attack()`（node-node 下自动触发）：  
   - 构造 member/non-member softlabel 差分范数特征（`gif.py:171`、`gif.py:188`）。  
   - `roc_auc_score` 得到 AUC（`gif.py:190`），写入 `average_auc`（`gif.py:193`）。  
7. **输出指标**：  
   effectiveness=`average_f1`，efficiency=`avg_training_time + avg_unlearning_time`，privacy=`average_auc`（`IF_based_pipeline.py:145`）。  

### 3) GNNDelete（learning-based）
**为什么补它**：这是 learning/distillation 家族里最“工程化”的删除模块实现，node/edge/feature 三分支都比较完整。  

1. `run_exp()` 走 `pipeline/Learning_based_pipeline.py:80`，每轮调用 `determine_target_model → train_original_model → unlearning_request → unlearn`。  
2. `determine_target_model()`：针对 `SGC/S2GC/SIGN` 先算 `features_pre`，再设 `GNNDeleteTrainer`（`gnndelete.py:65`、`gnndelete.py:75`）。  
3. `train_original_model()` 训练原模型并缓存 `original_softlabels`，给后续 MIA 用（`gnndelete.py:82`、`gnndelete.py:91`）。  
4. `unlearn()` 按任务分发到 `delete_node/delete_edge/delete_feature`（`gnndelete.py:96`、`gnndelete.py:102`）。  
5. `delete_node()`：读取 `unlearning_nodes`，构造 `df/dr/sdf` 掩码与 1-hop/2-hop 邻域（`gnndelete.py:383`、`gnndelete.py:439`、`gnndelete.py:427`）。  
6. 仅优化 deletion 参数（`named_parameters` 里 `del*`），再走 `gnndelete_train`（`gnndelete.py:469`、`gnndelete.py:515`）。  
7. 效果指标从 `eval_del('test')` 出来（`gnndelete.py:540`），隐私指标用 softlabel 位移 + `roc_auc_score`（`gnndelete.py:553`、`gnndelete.py:574`、`gnndelete.py:576`）。  
8. edge/feature 分支同理，入口分别在 `delete_edge` 与 `delete_feature`（`gnndelete.py:579`、`gnndelete.py:713`）。  

## 评估与攻击（MIA / poisoning / robustness）

1. **MIA 模块位置与 I/O**
   - 通用影子模型攻击：`attack/MIA_attack.py:24`（shadow model），`attack/MIA_attack.py:84`（attack model），输入是 soft labels，输出 AUC（`attack/MIA_attack.py:181`）。  
   - GraphEraser 专用：`attack/Attack_methods/GraphEraser_MIA.py:418`，输入是 unlearned/test 节点的后验，输出 `average_auc[self.run]`（`GraphEraser_MIA.py:507`）。  
   - GNNDelete 专用：`attack/Attack_methods/GNNDelete_MIA.py:4`，输出成员推断成功率。  

2. **AUC≈0.5 在代码里的含义**
   - 这里多数方法把“member vs non-member”转成二分类标签 `1/0`，再用 `roc_auc_score(label, score)`（如 `gif.py:190`、`GraphEraser_MIA.py:531`）。  
   - 当 score 对两类不可分时，AUC 会逼近 0.5，即“攻击接近随机猜测”。  

3. **poisoning / robustness 在本库中的实现现状**
   - 统一开关是 `--poison`（`parameter_parser.py:69`）。  
   - 主要行为是 edge unlearning 时注入负采样“毒边”（`utils/dataset_utils.py:101`、`utils/dataset_utils.py:107`）。  
   - 许多方法在 unlearn 前记录 poisoned baseline（`poison_f1`），如 GraphEraser `grapheraser.py:272`、GIF `gif.py:404`、GNNDelete `gnndelete.py:88`。  
   - **没有一个统一的“鲁棒性攻击 benchmark 调度器”**；更多是方法内部分支 + 指标记录。  

4. **最小可复现实验命令（含 MIA）**
```bash
python main.py --cuda 0 --dataset_name cora --base_model GCN --unlearning_methods GIF --unlearn_task node --downstream_task node --is_transductive True --is_balanced False --unlearn_ratio 0.1 --num_runs 1 --num_epochs 100
```
- `--unlearning_methods GIF`：走 `UnlearningManager.method_map["GIF"]`（`unlearning_manager.py:35`）。  
- `--base_model GCN`：`model_zoo.init_model()` 选 `GCNNet`（`model/model_zoo.py:60`）。  
- `--is_transductive/--is_balanced`：决定 split 缓存路径与 unlearning 文件根路径（`config.py:53`、`config.py:58`、`utils/dataset_utils.py:53`、`utils/dataset_utils.py:71`）。  
- `--unlearn_ratio 0.1`：控制 unlearning 节点采样比例（`gif.py:613`）。  
- `--unlearn_task node --downstream_task node`：触发 GIF 的 node request + `mia_attack()`（`gif.py:607`、`IF_based_pipeline.py:139`）。  

## 我下一步建议的阅读/实验清单（5-8条）
1. 先跑一遍上面的 GIF 命令，确认 `log/GIF/cora/GCN/*.log` 里 `F1/AUC/time` 三组指标。  
2. 再跑 GraphEraser 同配置对比效率：`--unlearning_methods GraphEraser --num_shards 10 --partition_method lpa_base`。  
3. 对 GraphEraser 切换 `--aggregator mean|optimal`，观察 `analysis_data/optimal` 与最终 F1 差异（`aggregator.py:121`）。  
4. 若你关心“遗忘彻底性”，优先查 affected shard 之外是否保留可识别后验（`grapheraser.py:712`）。  
5. 对 GIF 重点看 `iteration/damp/scale` 对速度和精度的 tradeoff（`gif.py:761`）。  
6. 若要做 GraphEraser-MIA 主流程复现，先确认是否要启用 `attack_unlearning()` 调用（当前基类里被注释，`pipeline/Shard_based_pipeline.py:175`）。  
7. 跑一轮 GNNDelete（`--unlearning_methods GNNDelete`）对比 deletion-parameter 路线与 MIA AUC。  

## 需要你决定/确认的点（我基于当前代码的最佳推断）
- **GraphRevoker 映射**：看起来应是 `graphrevoker` 类，但目前映射到 `grapheraser`；需运行一次 `--unlearning_methods GraphRevoker` 验证是否为有意复用。  
- **GraphEraser 的 MIA 默认是否开启**：新 pipeline 默认不调 `attack_unlearning()`；若你目标是完整“unlearning+MIA”，建议确认是否修改 `pipeline/Shard_based_pipeline.py:175`。  
- **`results/` 是否纳入主实验输出**：当前 `main.py` 主链未写 `results/`；你若要统一汇总 JSON，需额外接入现有 `results/step0_validation` 脚本。
