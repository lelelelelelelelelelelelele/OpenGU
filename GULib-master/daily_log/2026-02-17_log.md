### 2. Step 1: 攻击策略基础架构（下午）

#### 2.1 Base Strategy ABC
- **文件**: `attack/attack_strategies/base_strategy.py` (39行)
- **设计**:
  ```python
  class BaseStrategy(ABC):
      @abstractmethod
      def select_nodes(self, data, model, k) -> Tensor:
          """选择k个节点用于遗忘攻击"""
          pass

      def select_nodes_by_ratio(self, data, model, ratio) -> Tensor:
          """按比例选择节点"""
          k = int(data.num_nodes * ratio)
          return self.select_nodes(data, model, k)
  ```
- **设计决策**: 所有策略统一返回Tensor，支持k和ratio两种接口

### 4. Step 5: TracIn策略实现（晚上）

#### 4.1 TracIn算法原理
- **核心思想**: 用梯度相似度近似 influence function
- **公式**: `score_i = -sum_j (grad_i · grad_j)`
- **优势**: 无需计算Hessian，计算复杂度更低
- **参考**: Pruthi et al., NeurIPS 2020

#### 4.2 实现细节
- **文件**: `attack/attack_strategies/tracin_strategy.py` (173行)
- **关键组件**:
  - `_compute_node_gradient()`: 单节点梯度计算
  - `_compute_tracin_scores()`: 成对梯度点积求和
  - `_compute_node_loss()`: 支持cross_entropy/mse
- **梯度计算**: 使用`loss.backward()` + 参数梯度收集
- **设备处理**: 自动检测cuda/cpu

### 决策2: TracIn而非完整IF
- **选择**: 使用梯度相似度近似，而非Hessian-based IF
- **理由**:
  - 计算效率: O(N²·d) vs O(N·d² + d³)
  - 实现复杂度: 无需二阶导数和Hessian求逆
  - 论文支持: TracIn在CV/NLP领域已被验证有效

### 决策3: 确定性的Baseline
- **选择**: Degree和PageRank都是确定性算法
- **好处**: 实验可复现，无需多次运行取平均

### 发现2: TracIn计算成本
- **梯度计算**: 每个节点需要一次forward+backward
- **成对点积**: O(N²) 复杂度，N=1000时需要1e6次点积
- **优化空间**: 可改用batch计算或采样近似

## 💡 经验总结

### 1. PyG PageRank坑
- **问题**: `compute_pagerank` 返回的tensor需要detach
- **解决**: 在`select_nodes`中确保返回CPU tensor

### 2. 梯度计算陷阱
- **问题**: `model.zero_grad()` 必须在每次backward前调用
- **后果**: 不清理梯度会导致累积，TracIn分数错误

### 3. 测试驱动开发
- **做法**: 每实现一个策略，立即写5个基础测试
- **好处**: 提前发现边界问题（如空图、k>N等）

### 4. 接口一致性
- **决策**: 所有策略统一返回`torch.Tensor`而非`list`
- **好处**: 下游代码无需类型判断，可直接用于PyG索引
