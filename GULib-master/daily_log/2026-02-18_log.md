### 1. Bug 修复：demo_attack.py 参数传递

**问题**: `_extract_demo_args()` 使用 `parse_known_args()` 消费了 `--cuda`, `--dataset_name`, `--base_model` 等共享参数，导致后续 `parameter_parser()` 拿不到这些值，全部回退默认（如 `cuda=3`）。

**修复**: 在参数覆盖区域补全所有共享参数：
```python
args['cuda'] = demo_args.cuda
args['dataset_name'] = demo_args.dataset_name
args['base_model'] = demo_args.base_model
args['unlearning_methods'] = demo_args.unlearning_methods
args['unlearn_ratio'] = demo_args.unlearn_ratio
```

同时清理了两处 DEBUG 打印语句。

### 2. Bug 修复：pipeline_adapter.py 设备不匹配

**问题**: `_evaluate_model()` 中 model 在 GPU 但 `data.x`, `data.edge_index` 在 CPU，导致 random/degree/pagerank 三个策略全部报错：
```
RuntimeError: Expected all tensors to be on the same device, but found at least two devices, cpu and cuda:0!
```

**修复**: 在 `_evaluate_model()` 开头获取 model 所在设备并移动 data：
```python
device = next(model.parameters()).device
data = self.data.to(device)
```

### 3. Bug 修复：F1 baseline 记录缺失

**问题**: `gnndelete.py` 中 `poison_f1`（训练后、遗忘前的 F1）仅在 `args["poison"] and args["unlearn_task"]=="edge"` 时记录，node task 时 f1_before 始终为 0。

**修复**: 移除条件判断，始终记录 `poison_f1`：
```python
# Before: if self.args["poison"] and self.args["unlearn_task"]=="edge":
# After: Always record
self.poison_f1[self.run] = self.target_model.evaluate()
```

同步更新 `pipeline_adapter.py` 的 `run_with_strategy()` 从 `method.poison_f1` 提取 f1_before。

### 4. TracIn 性能优化

**文件**: `attack/attack_strategies/tracin_strategy.py`

**优化 1 — 矩阵运算替代嵌套循环**:
```python
# Before: O(N^2) Python for loops
for i, grad_i in enumerate(grads):
    for grad_j in grads:
        score -= torch.dot(grad_i, grad_j)

# After: 矩阵运算
G = torch.stack(grads)       # [N, d]
col_sum = G.sum(dim=0)       # [d]
scores = -(G @ col_sum)      # [N]  — 等价于 -(G @ G^T).sum(dim=1)
```

**优化 2 — train_mask 过滤**:
```python
# Before: 对全图 2708 个节点计算梯度
candidates = torch.arange(data.num_nodes)

# After: 只对训练集 ~140 个节点计算
candidates = data.train_mask.nonzero(as_tuple=False).squeeze(-1)
```

**效果**: 梯度计算从 2708 次 → ~140 次 backward pass，减少 ~95%。

## 关键发现

### 发现 1: TracIn 有效性确认
TracIn 基于梯度相似度选择节点，在 GNNDelete 上实现 1.32x 攻击增强。说明梯度影响力信号能有效识别对模型性能更关键的节点。

### 发现 2: 结构启发式失效
Degree 和 PageRank 均不如 Random（0.78x）。高度数/高 PageRank 节点不一定是 GNNDelete 的软肋——GNNDelete 通过 deletion_weight 参数学习遗忘，结构重要性 != 遗忘脆弱性。

### 发现 3: TracIn 耗时可控
节点选择耗时 8.63s（140 个 train_mask 节点），unlearning 本身 ~0.8s。优化后的矩阵运算使 TracIn 在 Cora 规模上完全可行。
