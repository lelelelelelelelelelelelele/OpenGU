# Step 0 验证结果 - 完整指标说明

## 回答：除了F1，还有哪些指标？

根据对所有11个方法的日志分析，以下是**可用的评估指标汇总**：

### 1. ✅ **F1 Score** (所有方法都有)

- **F1_before**: 删除前的 F1 分数（基础训练后）
- **F1_after**: 删除后的 F1 分数（unlearning 后）
- **可用于**: 所有11个方法
- **说明**: 这是最核心的性能指标，表示分类准确性

### 2. ⚠️ **Accuracy & Recall** (仅 GNNDelete 有)

- **Accuracy**: 测试集准确率
- **Recall**: 召回率
- **可用于**: 仅 GNNDelete 方法在日志中明确报告
- **说明**: 其他方法在训练过程中可能计算了，但未输出到日志

### 3. ⚠️ **AUC Score** (4个方法有有效值)

| 方法 | AUC 值范围 | 说明 |
|------|-----------|------|
| **GUIDE** | 0.9853 (所有比例都一样) | 非常高，表示成员推断攻击成功率高 |
| **GIF** | 0.8166 → 0.5194 (随删除比例下降) | 中等到低 |
| **GNNDelete** | 0.9704 → 0.5476 (随删除比例下降) | 高到中等 |
| **IDEA** | 0.6213 → 0.5312 (随删除比例下降) | 中等 |
| 其他7个方法 | 0.0000 | 未评估 MIA |

- **说明**: AUC = Membership Inference Attack AUC，衡量攻击者通过模型推断训练数据成员身份的能力
- **0.5 = 随机猜测**，越高表示模型泄露训练数据信息越多

### 4. ✅ **Time Metrics** (所有方法都有)

#### 4.1 Training Time (训练时间)
- **可用于**: 所有方法
- **范围**: 0.000s ~ 0.942s
- **说明**: 基础模型训练耗时

#### 4.2 Unlearning Time (遗忘时间)
- **可用于**: 所有方法
- **范围**: 0.343s ~ 10.613s
- **说明**: 执行 unlearning 操作的耗时
- **最快**: MEGU (~0.35s), GIF (~0.40s)
- **最慢**: GraphRevoker (~10s), GUIDE (~8s)

#### 4.3 Partition Time (分区时间)
- **可用于**: 仅 shard-based 方法 (GraphEraser, GUIDE, GraphRevoker)
- **范围**: 2.88s ~ 3.48s
- **说明**: 图分区操作的耗时

#### 4.4 Sampling Time (采样时间)
- **可用于**: 仅部分 learning-based 方法
- **SGU**: 0.197s (唯一报告了非零值的)
- **其他**: 0.000s 或未报告

### 5. ❌ **Loss** (大部分方法未报告)

- **可用于**: 仅 GNNDelete 在日志中明确输出最终 loss
- **说明**: 其他方法在训练过程中计算了 loss，但未输出到 Performance Metrics 部分

### 6. ❌ **Poison F1 Score** (所有方法都是 0.0000)

- **说明**: 当前实验未启用 poison attack，所以都是 0

## 关键发现

### 1. **F1 是最全面的指标**
所有方法都报告了 F1 Score，这是评估 unlearning 效果的主要指标。

### 2. **Accuracy/Recall 数据稀缺**
只有 GNNDelete 明确输出了这些指标。如果需要这些指标，可能需要修改代码在其他方法中也输出。

### 3. **AUC 评估不完整**
- 只有 4/11 方法报告了 MIA AUC
- 其他 7 个方法都是 0.0000，说明未启用 MIA 评估
- **GUIDE 的 AUC=0.9853 异常高**，可能是因为它本质上在做 retrain

### 4. **Time 指标最完整**
所有方法都报告了 Training Time 和 Unlearning Time，这对性能分析很有用。

### 5. **Projector 数据缺失**
Projector 方法连 F1_after 都是 N/A，说明可能有 bug 或日志输出问题。

## 建议

### 如果需要更多指标：

1. **修改代码输出 Accuracy/Recall**
   - 在 `task/BaseTrainer.py` 或各方法的 trainer 中添加输出
   - 在 Performance Metrics 部分添加这些字段

2. **启用 MIA 评估**
   - 在所有方法中统一启用 AUC 计算
   - 修改 `attack/MIA_attack.py` 相关代码

3. **添加更多分类指标**
   - Precision
   - ROC-AUC (分类任务的 AUC，不是 MIA 的 AUC)
   - Confusion Matrix

4. **添加图结构指标**
   - 删除后的图连通性变化
   - 节点度分布变化
   - 聚类系数变化
