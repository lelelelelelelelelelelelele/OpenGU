# AI 论文协作与生成指南 (AI Paper Generation Guide)

你好，AI 助手！当你被要求在此文件夹（`report/paper/`）下协助生成或细化学术论文的大纲、正文草稿或结论时，你必须作为一名严谨的学术协作者。请优先遵循以下指南来获取准确的项目上下文、实验数据与参考文献，严禁凭空捏造（Hallucination）。

## 1. 📖 背景知识与核心概念 (Background & Concepts)
为了能够准确理解本项目的目标（如机器视觉/大模型的遗忘学习 Unlearning）、相关算法与具体实现机制，请优先读取以下文件：
- **项目主控文档**：读取根目录的 `../../README.md` 与 `../../CLAUDE.md` 以获取项目的宏观架构、核心知识点、指令规范以及整体使用方式。
- **当前任务与进展**：通过查看 `../../todo.md` 以及上级目录中的每日记录 `../daily-log/`，了解我们目前正在攻克的关键技术瓶颈和创新点。
- **工作流设定**：可参考并读取工作流文件 `../report_workflow.md` 以确保符合我们预设的报告/论文撰写流程。

## 2. 📊 实验结果查询 (Experimental Results)
在构建“实验设置 (Experimental Setup)”或“实验结果与分析 (Results and Analysis)”章节时，请务必基于实际跑出的量化数据进行分析：
- **核心 Benchmark 数据**：优先级最高的分析数据位于 `../../experiments/im_benchmark/bench_results.json`，请读取该 JSON 获取指标对比（如 Accuracy, Robustness, 遗忘效率 等）并据此辅助作图或绘制表格。
- **详细运行输出**：其它细粒度生成数据（防御、攻击评估、生成的图像等）位于 `../../results/` 目录下。
- **运行日志与参数**：可以参考 `../../log/` 里的日志文本，或者查看具体的实验启动脚本如 `../../experiments/im_benchmark/run_benchmark.py` 来搞清楚实验的参数和设定。

## 3. 📚 参考文献与先验文献 (References & Literature)
在撰写 “引言 (Introduction)” 与 “相关工作 (Related Works)” 部分时，必须做到有据可查：
- **项目文献库**：请检索并查阅 `../../papers/` 目录，这里存放了大量本项目所参考的前沿学术文献资源（PDFs或笔记）。
- **引用文件**：如果用户在此 `report/paper/` 目录下放入了 `.bib` 等参考文献文件，请在正文中严格按照其规范的 ID 插入引用（例如：`\cite{xxx}`）。

## 4. 💻 代码与方法论实现细节 (Implementation Data)
当需要撰写 “方法论 (Methodology)” 章节并提供图表/公式/伪代码解释时，你需要直接查阅项目源代码来保证完全一致：
- **遗忘学习核心方法**：请查阅 `../../unlearning/` 目录下的核心逻辑与 `../../pipeline/` 中的工作流。
- **评估/攻击机制**：查阅 `../../attack/` 以及 `../../eval_collateral.py` 等脚本。
- **数据集于模型配置**：查阅 `../../dataset/` 与 `../../model/`，以便在论文中准确描写所使用的数据增强与网络架构。

## 🎯 AI 助手工作流操作建议（推荐给用户的执行顺序）

1. **Step 1 (读取指令)**：用户输入撰写需求时，AI 首先读取本 `README.md` 唤醒对项目的整体记忆与文件位置定位。
2. **Step 2 (数据搜集)**：AI 主动调取上述提到的文件（如 `bench_results.json` 和 `CLAUDE.md` 等），形成初始知识上下文。
3. **Step 3 (建立大纲)**：根据获取的背景和数据，立刻为用户输出一份具备“标题、拟写内容说明、所需图表位置”的骨架级论文大纲。
4. **Step 4 (逐节推进)**：与用户交互，确认大纲无误后，结合具体子目录或代码的二次深度检索，逐一 Section 填充有学术深度的段落草稿。

*注意：对于缺乏数据支撑的论点，AI 应当主动标记为`[待补充]`并提问用户，而非自行推测测试结果。*
