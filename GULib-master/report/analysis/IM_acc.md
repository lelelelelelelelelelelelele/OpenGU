# 📋 IM 选点策略性能与精度对比实验方案

## 1. 实验目的
针对 CELF 算法在低传播概率（$p=0.1$）下退化及 Python 循环开销导致的性能瓶颈，实现并对比多种优化方案。
在保证最终选点集合的高质量（与原方案高重叠率、最终 Influence Spread 接近）的前提下，寻找最适合大数据集的极速选点策略。

## 2. 实验设置
*   **测试数据集**: Cora (Nodes: 2708, Edges: ~10556) —— *可扩展至 PubMed 等更大数据集*
*   **超参数**: 
    *   目标选点数 $k = 135$
    *   传播概率 $p = 0.1$
    *   MC 模拟次数 `mc_samples = 100`
*   **核心评估指标**:
    1.  **耗时 (Execution Time)**: 算法运行总秒数。
    2.  **重叠率 (Jaccard Similarity / Intersection Size)**: 与 Baseline 选出节点集合的交集大小，公式：$|S_{opt} \cap S_{base}| / k$。
    3.  **最终真实传播范围 (Final Spread)**: 使用选出的节点集合 $S$，统一运行一次独立的 10,000 次 MC 模拟，评估其真实影响力（这最重要，因为即便节点不同，只要 Spread 一样，优化就是成功的）。

## 3. 待实现的变体方案 (Agent 任务列表)

请 Agent 在 `im_strategy.py` 或新建的 `im_experiment.py` 中实现以下 5 个版本的 IM 逻辑：

### V0: Baseline (当前原版 CELF)
*   **逻辑**: 保留当前所有逻辑，包括 Step 1 的初始计算和 Step 2 的 Lazy Greedy 循环。
*   **用途**: 作为耗时、重叠率和 Spread 的基准线。

### V1: Fast-Hash CELF (仅修复 Python 性能 Bug)
*   **逻辑**: 算法逻辑与 V0 完全一致，**仅优化 Python 层的缓存 Key 生成**。
*   **Agent 修改指令**:
    找到 Step 2 循环中的这行代码：
    `base_seed = self.random_seed * 10000 + int(np.sum(np.sort(seed_arr))) % 10000`
    替换为无排序、无 numpy 数组分配的极速版本（利用求和交换律）：
    `base_seed = self.random_seed * 10000 + sum(selected_set_list + [node]) % 10000`
*   **预期**: 重叠率 100%，测试纯 Python 开销占总耗时的比例。

### V2: Top-K Initial Gain (零贪心/一次性选点)
*   **逻辑**: 仅执行原 CELF 的 Step 1。计算所有候选节点的初始单节点 Spread，直接降序排列，取 Top-$k$ 作为最终结果，完全跳过 Step 2 的 `while len(S) < k` 循环。
*   **预期**: 速度极快（~5s），需观察在 $p=0.1$ 时其精度损失有多大。

### V3: Candidate Pruning CELF (候选集剪枝)
*   **逻辑**: 利用图的结构特征缩小候选池。在进入 CELF Step 1 之前，计算所有节点的 Degree（度数），只保留 Degree 排名前 $M$ 的节点（例如 $M = 3 \times k \approx 400$）作为候选节点，然后运行标准的 CELF（V1 版本）。
*   **预期**: 极大减少 Step 1 的耗时和 Step 2 的堆大小，预期重叠率 > 90%。

### V4: Batch CELF (批量贪心)
*   **逻辑**: 减少 Step 2 的循环轮数。在 Step 2 的 Lazy Evaluation 中，每次确认了堆顶节点是最优后，不仅将其加入集合，而是连续弹出堆顶的 $B$ 个节点（例如 $B=5$ 或 $10$），直接加入选中集合。
*   **Agent 修改指令**: 将外层循环从每次加 1 个点，改为每次验证并加入 $B$ 个点，直到满 $k$ 个。
*   **预期**: 速度提升 $B$ 倍，需验证批量加入带来的次优选择对最终 Spread 的影响。

### 你拿到结果后如何决策？
1.  **如果 V1 耗时降到了 50 秒以内**：说明纯粹是 `np.sort` 的锅，你直接用 V1 即可，无需牺牲任何精度。
2.  **如果 V2 重叠率 > 90% 或 最终 Spread 损失 < 2%**：果断在全局采用 V2 (Top-K)，因为它最简单、最快，非常适合大图。
3.  **如果 V2 效果不好，但 V3 表现优异**：在大数据集上采用 V3（度数剪枝 + CELF），兼顾了集合覆盖的严谨性和运行性能。
