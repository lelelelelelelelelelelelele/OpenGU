## 已完成工作

### 1. CELF Numba 加速 (im_strategy.py)

将 IM 策略的 Monte Carlo BFS 模拟从纯 Python 升级为 numba JIT 编译，实现 6 项优化：

| 优化项 | 内容 |
|--------|------|
| 稳定哈希 | `base_seed = 2024 * 10000 + sum(sorted(seed_set)) % 10000`，避免 `hash()` 跨进程漂移 |
| Stamp-visited | 递增 stamp 标记替代每轮 O(N) 清零，PubMed 省 ~15GB 内存写入 |
| Splitmix64 RNG | 内联自定义 RNG，无全局状态，比 `np.random` 快 2-3x |
| CSR 确定性 | `lexsort((dst, src))` + 去重，确保邻接遍历顺序稳定 |
| int32 dtype | indices/frontier/visited 全用 int32，省一半内存 |
| JIT cache | `@numba.njit(cache=True)` 持久化编译，首次 ~2-3s，后续跳过 |

**新增功能**:
- `HAS_NUMBA` flag + graceful fallback（无 numba 时自动退回纯 Python）
- `candidate_fraction` 参数（大规模图候选集裁剪）
- `_build_adjacency_csr()` CSR 构建
- `_prune_candidates_by_degree()` 度裁剪

**文件**: `attack/attack_strategies/im_strategy.py` (+278 行, -22 行)

## 关键发现

### Collateral Damage 三方法对比（Cora/GCN, ratio=0.05）

#### GNNDelete — 攻击最有效

| Strategy | Gap% | MeanShift | Flipped% |
|----------|------|-----------|----------|
| random | 6.86% | 0.273 | 11.67% |
| **degree** | **29.48%** | **0.559** | **33.35%** |
| pagerank | 21.44% | 0.397 | 23.47% |
| tracin | 11.02% | 0.284 | 12.88% |
| **im** | **24.02%** | 0.359 | **26.54%** |
| hybrid | 10.83% | 0.159 | 10.14% |

**洞察**: Degree 策略对 GNNDelete 造成 29.5% 的 Gap，说明 GNNDelete 对高度节点删除极度敏感。

#### GIF — 高度鲁棒

| Strategy | Gap% | MeanShift | Flipped% |
|----------|------|-----------|----------|
| random | -0.42% | 0.015 | 1.26% |
| degree | 1.23% | 0.016 | 1.31% |
| pagerank | 0.82% | 0.015 | 0.97% |
| tracin | 0.62% | 0.014 | 1.12% |
| im | 0.21% | 0.015 | 1.28% |
| hybrid | 0.83% | 0.017 | 0.94% |

**洞察**: GIF 几乎不可攻击，所有策略 Gap < 1.3%，collateral damage 极低。

#### GraphEraser — 高 collateral, 中等 Gap

| Strategy | Gap% | MeanShift | Flipped% |
|----------|------|-----------|----------|
| random | 0.94% | 0.237 | 21.78% |
| degree | 6.70% | 0.231 | 22.56% |
| pagerank | 6.83% | 0.236 | 22.06% |
| tracin | -0.96% | 0.234 | 21.27% |
| im | 4.84% | 0.244 | 23.09% |
| hybrid | 0.24% | 0.232 | 20.97% |

**洞察**: GraphEraser 自身 collateral damage 极高（~21% flipped 无论策略），但 Gap 远小于 GNNDelete。Shard-based 方法天然不精确但对攻击有一定抵抗力。

### 核心结论

1. **GNNDelete 是最脆弱的方法** — degree/im/pagerank 均可造成 20-30% Gap
2. **GIF 几乎免疫** — 影响函数方法鲁棒性极强
3. **GraphEraser 的 collateral damage 是结构性的** — 与攻击策略无关，~21% 始终存在
4. **Degree 策略意外强大** — 在 GNNDelete 上超过 TracIn 和 IM
5. **归因框架验证**: GNNDelete Gap >> 0 支持 H1（近似误差可被利用），GIF Gap ≈ 0 支持 H0

## 经验总结

1. **numba + CSR 是图算法加速的标准方案** — splitmix64 内联 RNG 避免全局状态是关键
2. **测试 numba vs Python 一致性需要结构化图** — 随机图上节点 spread 差异太小，rank correlation 不可靠；用有向星图测 argmax 更稳定
3. **设备不匹配是 PyG pipeline 的常见 bug** — 不同 pipeline 返回模型的设备不一致，`_predict()` 层统一处理是正确策略
4. **cache 匹配需注意 unlearn_ratio** — 默认 0.1 vs 实验用 0.05，cache miss 导致 skip
