- 所有策略对 GUIDE 的 F1 Drop ≈ 0.0000——**不是因为 GUIDE 免疫攻击**
- **node 任务 Bug**：`guide.py` 缺少 `else` 分支，`poison_f1`（f1_before）始终为 0；`pipeline_adapter` fallback 导致 f1_before = f1_after → F1 Drop = 0
- **edge 任务 Bug**：将 `AUC_score` 误存入 `average_f1` 和 `poison_f1`，F1 指标实为 AUC 值
- **所有 GUIDE 历史实验结果均无效，需修复源库后系统重做**
- 修复后 GUIDE 的真实攻击效果未知，待实验重新评估

### 3. GNNDelete 极度脆弱（重要发现）
- ratio=0.01（仅 27 节点）时 F1 Drop 可达 17.34%（hybrid_v4）
- collateral gap 高达 20%+，意味着非目标节点也受到严重影响
- 对比 GIF（collateral gap ≈ 0-2%），GNNDelete 的近似误差极大
